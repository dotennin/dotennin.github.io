オンプレミスからAWSへの移行についての総合問題

インポートの可否

オンプレミス側で使用している製品はVMware ESX および Microsoft Hyper-V の2種類です。この2種類の仮想マシンイメージをAWS上にインポートする予定です。どの機能を使用すればインポートできますか？最も適したものを一つ選択してください。

問題1
　
A

B
AMI Import/Export
C

VM Import/Export
問題 1 の説明および補足 
再構築の手間なく簡単にAWSへ移行するにはVM Import/Exportという機能を使います。イメージをそのままAmazon EC2インスタンスに変換して起動させることが可能です。サポートされる仮想マシンイメージは、VMware ESXやVMWare vSphereで利用されるVMDK、Microsoft Hyper-VやCitrix XenServerで利用されるVHDになります。

インポート時の注意点（Windows）

仮想マシンイメージをインポートするときに、さまざまな理由でインポートが失敗する場合や想定した動作を行わないことがありました。
ログインできない
インポートされたWindowsインスタンスにログインすることができない事象が発生しました。解決策について説明しているものを一つ選択してください。

問題1
　
A
VM Import/Exportのオプションでリモートデスクトップを有効にする
B
リモートデスクトップ（RDP）によるリモート管理を有効にしておく
問題 1 の説明および補足 
Windowsでリモートデスクトップによるリモート管理を有効にしていないと、インポート完了後にEC2インスタンスにログオンすることができません。また、Windows FirewallでRDP（TCP:3389）のポートをブロックしないに設定しておきます。ブロックしているとEC2インスタンスに接続することができません。AWS側はSecurity Groupの設定でTCP:3389を開ける必要もあります。
