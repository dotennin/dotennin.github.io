AWS Storage Gateway の特徴

Storage Gateway の紹介

AWS Storage Gateway は、既存のオンプレミス環境と AWS クラウドを接続するハイブリッドクラウドストレージサービスです。この機能を使用すると、クラウドバックアップを使い始める、オンプレミスマシンで生成されたデータのワークフローをクラウドで処理する、ブロックボリュームのデータやデータベースを一括移行するといった、クラウド導入のあらゆる段階においてハイブリッドクラウドワークロードを簡単に実行できます。

 

主な特徴

標準ストレージプロトコル。 Storage Gateway は、NFS、SMB、iSCSI、iSCSI-VTL を使用して、ローカルの本番アプリケーションやバックアップアプリとシームレスに接続します。このため、アプリケーションに修正を加えることなく、AWS クラウドストレージを導入できます。プロトコル変換とデバイスエミュレーションにより、Amazon S3 上の Storage Gateway で管理されるボリュームのブロックデータにアクセスしたり、ファイルをネイティブの Amazon S3 オブジェクトとして保存したりできます。また、S3 に保存されている仮想テープライブラリ内で仮想テープバックアップをオンラインで保管したり、このバックアップを Amazon Glacier のテープアーカイブ層に移行することもできます。

フルマネージド型のキャッシュ。 このローカルのゲートウェイアプライアンスでは、最近読み書きされたデータのキャッシュを保持します。このため、アプリケーションから、AWS に永続的に保管されたデータへの低レイテンシーのアクセスが可能になります。ゲートウェイではリードスルーおよびライトバックのキャッシュを使用しています。

最適化されたセキュアなデータ転送。Storage Gateway では、SSL を使用してあらゆるゲートウェイアプライアンスと AWS 間の転送データを暗号化することで、変更したデータのセキュアなアップロードと、リクエストしたデータのセキュアなダウンロードが可能です。マルチパート管理、自動バッファリング、デルタ転送といった最適化がゲートウェイのすべてのタイプにおいて使用され、すべてのブロックと仮想テープのデータに対して圧縮が適用されます。

AWS との統合。 AWS のネイティブサービスである Storage Gateway は、ストレージ、バックアップ、管理のための AWS の他のサービスと統合されています。Storage Gateway では、ファイルをネイティブの Amazon S3 オブジェクトとして保存したり、仮想テープを Amazon Glacier にアーカイブしたりできます。また、ボリュームゲートウェイで生成された EBS スナップショットを Amazon EBS で保存することもできます。また、AWS Backup との統合により、バックアップの管理やボリュームゲートウェイのボリューム復旧を行えるため、バックアップ管理を簡素化し、ビジネスおよび規制関連のバックアップに関するコンプライアンス要件を満たすことができます。 

さらに、Storage Gateway ではオンプレミスゲートウェイと AWS のサービスの両方に対して AWS コンソールを使用した一貫性のある管理環境を提供しています。こうした AWS のサービスには、モニタリング、管理、セキュリティ向けに用意された Amazon CloudWatch、AWS CloudTrail、AWS Identity and Access Management (IAM)、AWS Key Management Service (KMS) などが含まれます。

ゲートウェイのタイプ

ファイルゲートウェイ

ファイルゲートウェイのファイルインターフェイスでは、業界標準の NFS および SMB ファイルプロトコルを使用して Amazon S3 にオブジェクトとしてファイルを保存できます。保存したファイルには、NFS および SMB を介してデータセンターや Amazon EC2 からアクセスするか、S3 API を使用してオブジェクトとしてアクセスすることができます。POSIX 形式のメタデータ (所有権、権限、タイムスタンプを含む) は、Amazon S3 内のファイルに関連付けられたオブジェクトのユーザーメタデータに永続的に保管されます。S3 に転送されたオブジェクトは、ネイティブの S3 オブジェクトとして管理できるようになり、バージョニング、ライフサイクル管理、リージョン間レプリケーションなどのバケットポリシーは、バケットに保存されたオブジェクトに直接適用されます。

お客様はファイルゲートウェイを使用し、オブジェクトベースのワークロードで使用するためのファイルデータを S3 内に保存します。これには、データ分析や機械学習が含まれ、バックアップ向けのコスト効果の高いストレージターゲットや、アプリケーションのファイルストレージ向けにクラウド内でリポジトリや階層として利用できます。

ボリュームゲートウェイ

ボリュームゲートウェイでは、iSCSI プロトコルを使用しているブロックストレージボリュームをアプリケーションに提供します。このボリュームに書き込まれたデータは、ボリュームのポイントインタイムスナップショットとして非同期的にバックアップして、クラウド内に Amazon EBS スナップショットとして保存できます。オンプレミスのボリュームゲートウェイのボリュームをバックアップするには、サービスのネイティブスナップショットスケジューラを使用することも、AWS Backup サービスを使用することもできます。いずれの場合も、ボリュームバックアップは Amazon EBS スナップショットとして AWS に保存されます。このスナップショットは、変更されたブロックのみをキャプチャする増分バックアップです。すべてのスナップショットストレージも圧縮され、ストレージ料金も最低限に抑えられます。

iSCSI ブロックインターフェイスでボリュームゲートウェイに接続すると、キャッシュ型と保存型という 2 つのモードでゲートウェイを実行できます。キャッシュ型モードでは、プライマリデータを Amazon S3 に保存し、頻繁にアクセスするデータをキャッシュしてローカルに保持します。このモードを使用すると、プライマリストレージのコストを大幅に削減し、ストレージをオンプレミスでスケールする必要性を最小限に抑え、頻繁にアクセスするデータへの低レイテンシーのアクセスを維持できます。

保存型モードでは、データセット全体をローカルに保存し、Amazon S3 ボリュームの非同期コピーと EBS のポイントインタイムスナップショットを作成できます。このモードにより、耐久性が高くかつ低コストのオフサイトバックアップを実現でき、別のサイトに、または Amazon EC2 にローカルで復元できます。

多くの場合、ローカルアプリケーションをバックアップし、EBS スナップショットやキャッシュ型ボリュームのクローンに基づいた災害対策にそのバックアップを使用するために、ボリュームゲートウェイが活用されています。ボリュームゲートウェイと AWS Backup の統合により、お客様は AWS Backup サービスを利用することで、Storage Gateway のボリュームを使用するオンプレミスのアプリケーションを保護できます。AWS Backup では、キャッシュ型ボリュームと保存型ボリューム両方のバックアップと復元をサポートしています。AWS Backup とボリュームゲートウェイを使用すれば、バックアップ管理を一元化して運用上の負担を軽減し、コンプライアンス要件を満たすことができます。AWS Backup を使用すると以下のことが可能になります。

  • バックアップ基準を満たすカスタマイズ可能なスケジュールされたバックアップポリシーを設定できます。
  • 保持ルールと有効期限ルールを設定でき、ボリュームのポイントインタイムバックアップの手動管理やカスタムスクリプトを開発する必要がなくなります。
  • 統合されたビューによって、複数のゲートウェイや AWS の他のリソースのバックアップの管理とモニタリングが行えます。

テープゲートウェイ

テープゲートウェイは、仮想メディアチェンジャーと仮想テープドライブで構成される、業界標準の iSCSI ベースの仮想テープライブラリ (VTL) として、既存のバックアップアプリケーションに表示されます。ほぼ無制限の仮想テープの集合体に書き込んでいる間も、既存のバックアップアプリケーションとワークフローを継続して使用できます。それぞれの仮想テープは Amazon S3 に保管されます。仮想テープのデータに即時または頻繁にアクセスする必要がなくなった場合は、バックアップアプリケーションで仮想テープを Storage Gateway の仮想テープライブラリから Amazon Glacier クラウドストレージのアーカイブ層に移動することで、ストレージコストをさらに削減できます。

現時点で、Storage Gateway は、主要なバックアップアプリケーションと互換性があります。テープゲートウェイの VTL インターフェイスにより、テープオートメーションの多額の前払い資本支出、複数年のメンテナンス契約義務、継続的なメディアコストが不要になります。お支払いは使用する容量に対してのみです。オフサイト施設にストレージメディアを輸送することやテープメディアを手作業で取り扱うことは必要なくなり、AWS クラウドプラットフォームの設計と耐久性の利点を活用してアーカイブできます。

Storage Gateway のデプロイオプション

AWS Storage Gateway サービスは、クラウド内のコンポーネントで構成されます。これにはマネジメントコンソール、ストレージインフラストラクチャ、バックエンドのコントロール、統合サービス、API、およびアプリケーションにデプロイして接続するゲートウェイアプライアンスが含まれています。

デプロイ方法には、VMware ESXi で実行可能で Storage Gateway ソフトウェアが含まれている仮想マシン、オンプレミスのハードウェアアプライアンスとして、VMware Cloud on AWS の VM として、または Amazon EC2 の AMI としてのオンプレミスの Microsoft Hyper-V による、4 つのオプションがあります。

ハードウェアアプライアンスとしての Storage Gateway

Storage Gateway はハードウェアアプライアンスとして利用可能で、構成検証済みの Dell EMC PowerEdge R640XL サーバーに事前インストールされています。ハードウェアアプライアンスでは、仮想インフラストラクチャの制限、複雑に一元化されたリソースのプロビジョニングプロセス、IT 人員の制限があるお客様向けに、シンプルな調達、デプロイ、管理の環境を提供します。
