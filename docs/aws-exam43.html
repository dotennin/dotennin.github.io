AMI の所有者は、起動許可を指定することで可用性を決定することができます。起動許可の内、「すべての AWS アカウントに起動許可を与える」設定について、正しいものを一つ選択してください。

問題 1 の説明および補足 
「すべての AWS アカウントに起動許可を与える」設定は、パブリックが正解です。起動許可の指定は以下の3種類あります。

1.パブリック：所有者はすべての AWS アカウントに起動許可を与えます。
2.明示的：所有者は特定の AWS アカウントに起動許可を与えます。
3.暗示的：所有者には AMI の暗示的起動許可があります。
参考URL:起動許可
問題2
AWS OpsWorks と AWS CloudFormation の説明文について、空欄の箇所を埋めてください。
CloudFormation と AWS OpsWorks の違いは、 CloudFormation が AWS の各種サービスを組み合わせて使うためのテンプレートを用意して自動化できるツールなのに対し、 AWS OpsWorks は（＿＿＿＿）（レシピと呼ばれるRubyのコードで書かれた構築手順書）を用いて、自動化できるツールです。

問題 2 の説明および補足 
・AWS OpsWorks
Chef(レシピと呼ばれるRubyのコードで書かれた構築手順書)を実行するとその手順通りに自動構築してデプロイしてくれるサービスです。AWS CloudFormation に比べると、サポート範囲はアプリケーション志向の AWS リソースに限られています。例えば Amazon EC2 インスタンス、Amazon EBS ボリューム、Elastic IP、Amazon CloudWatch メトリクスなどです。
・AWS CloudFormation
CloudFormation が AWS の各種サービスを組み合わせて使うためのテンプレートを用意して自動構築してデプロイしてくれるサービスです。
・Elastic Beanstalk
アプリケーションコードを用意するだけでそのアプリを動かす環境を自動構築してデプロイしてくれるサービスです。
問題3
パブリックサブネットとプライベートサブネットを持つ VPC（NATインスタンス）で、データベース・サーバーはどこに起動したほうが良いですか。もっとも適したものを選択してください。
A
パブリックサブネット
B
どちらでも良い
C
プライベートサブネット
問題 3 の説明および補足 
データベース・サーバー は プライベートサブネット に配置します。

このシナリオの設定には、パブリックサブネットとプライベートサブネットを持つ VPC と、パブリックサブネットのネットワークアドレス変換（NAT）インスタンスが含まれます。NAT インスタンスにより、プライベートサブネットのインスタンスは、インターネットへのアウトバウンドトラフィックを開始できます。このシナリオは、パブリックにはアクセスできないバックエンドサーバーを維持しながら、パブリックなウェブアプリケーションを実行する場合にお勧めします。一般的な例としては、パブリックサブネットのウェブサーバーとプライベートサブネットのデータベースサーバーを持つ多階層のウェブサイトが挙げられます。ウェブサーバーがデータベースサーバーと通信できるように、セキュリティとルーティングを設定できます。
パブリックサブネットのインスタンスはインターネットから直接インバウンドトラフィックを受信できますが、プライベートサブネットのインスタンスはこれができません。また、パブリックサブネットのインスタンスはアウトバウンドトラフィックを直接インターネットに送信できますが、プライベートサブネットのインスタンスはできません。代わりに、プライベートサブネットのインスタンスは、パブリックサブネットの NAT インスタンスを使用して、インターネットにアクセスできます。
問題4
Amazon RDS で使用されているDB インスタンスの SSL 証明書は、2015 年 3 月 23 日よりAWSによって更新が開始されました。2015 年 4 月 3 日に古い証明書（rds-ca-2010）が期限切れになるために、新しく証明書を更新する必要があります。更新の必要性を確認するために、現在使用されている SSL 証明書 が 古い証明書（rds-ca-2010）になっているか確認することにしました。確認方法で正しいものを一つ選択してください。
A
DB インスタンスの詳細の [Security and Network] セクションの下の一覧表示より確認する
B
Amazon RDS コンソール より ナビゲーションペインで [DB Instances] を選択し[Instance Actions] ドロップダウンメニューから、[Modify] を選択して確認する
C
Amazon RDS コンソール より ナビゲーションペインの [Security Groups] をクリックし、利用可能な DB セキュリティグループの一覧表示より確認する
D
該当なし
問題 4 の説明および補足 
DB インスタンスの詳細の [Security and Network] セクション下の一覧表より確認できます。使用のインスタンスに "rds-ca-2015" と表示される場合は、新しい証明書が正常に適用されています。
参考URL：SSL 証明書の更新

問題7
Instance Store-Backed Linux AMI および Instance Store-Backed Windows AMI について正しいものを２つ選択してください。
A
Instance Store-Backed Windows AMI から Amazon EBS-Backed Windows AMI に変換できる
B
Instance Store-Backed Linux AMI は、Amazon EBS-Backed Linux AMI に変換できない
C
Instance Store-Backed AMI から Amazon EBS-Backed AMI を作成する際に、Amazon EBS ボリュームは、元のインスタンスストアのルートボリュームと同じサイズ以上である必要がある
D
Instance Store-Backed Linux AMI は、Amazon EBS-Backed Linux AMI に変換できる
問題 7 の説明および補足 
Instance Store-Backed Linux AMI は、Amazon EBS-Backed Linux AMI に変換できますが、Instance Store-Backed Windows AMI から Amazon EBS-Backed Windows AMI への変換、および所有していない AMI の変換はできません。
Instance Store-Backed AMI を Amazon EBS-Backed AMI に変換する際、Amazon EBS ボリュームを準備します。この Amazon EBS ボリュームは、元のインスタンスストアのルートボリュームと同じサイズ以上である必要があります。また、Instance Store-Backed AMI をバンドルするのに使用した X.509 プライベートキーをインスタンスにアップロードします。Amazon はこのキーを使用して、お客様と Amazon EC2 だけがお客様の AMI にアクセスできるようにします。
