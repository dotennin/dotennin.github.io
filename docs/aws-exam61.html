    ・パブリックサブネットのルーティング・テーブルを修正する
    ・追加したインスタンスのホストOSで、インターネット向けのルーティングを設定する
    したがって、追加したインスタンスにEIP(Elastic Ip Address) を割り当てる が正解になります。

    Category: Amazon RDS

    Amazon RDS の DBインスタンスを Multi-AZ 配備した場合のメリットについて正しいものを２つ選択してください。

    正解

    Multi-AZ 配備した場合は「プライマリ」と「スタンバイ」の二種類のDBインスタンスがあります。「プライマリ」はデータベースに読み込みと書き込みができます。「スタンバイ」はプライマリの最新レプリカになります。「スタンバイ」は「プライマリ」に障害やメンテナンスウィンドウの期間中であれば「プライマリ」に「昇格」します。「昇格」後は「プライマリ」になりデータベースに読み込みと書き込みを継続できます。またMulti-AZは同じ地域のデータセンターにDBインスタンスが配置されますが、違う地域のデータセンター間には配置されません。
    Multi-AZはリードレプリカに対応していないため、DB インスタンスの容量を伸縮自在に拡張して読み込み負荷を緩和することはできません。また、「プライマリ」と「スタンバイ」で「スタンバイ」は読み込みと書き込みをサポートしていないため、IOPSの向上は見込めません。

    不正解

    Category: IAM

    ユーザやグループ、ロール、リソースにアクセス許可を割り当てるにはポリシーを作成します。ポリシーはアクセス許可を明示的にリスト化したドキュメントです。基本的にポリシーによって指定することができます。ユーザーがアクセスを要求したときにどのような結果となるかを判断するかを示す、IAM ポリシーで使用できる エレメント を一つ選択してください。

    正解

    ユーザーがアクセスを要求したときにどのような結果となるかを判断する エレメント は「effect」になります。それぞれの意味は以下の通りです。
    Actions: 許可するアクション。各 AWS サービスには、それぞれ独自のアクションがあります。たとえば、ユーザーに対して Amazon S3 ListBucket アクションを使用することを許可できます。これはバケット内のアイテムに関する情報を返します。明示的に許可していないアクションはすべて禁止されます。
    Resources: アクションの対象として許可するリソース。たとえば、ユーザが ListBucket アクションをどの Amazon S3 バケットに対して実行してよいかを指定します。ユーザは明示的にアクセス許可を与えられていないリソースにはアクセスできません。
    Effect: ユーザーがアクセスを要求したときにどのような結果となるか（許可または拒否）。デフォルトでは、リソースに対するユーザーからのアクセスは拒否されるので、お客様はユーザーに対してリソースへのアクセスを許可することを明示的に指定する必要があります。

    ・Amazon S3 のバケットの中にバケットを作成することはできません。

    ・S3 Server Side Encryptionを使用して、Amazon S3 のオブジェクトを暗号化した状態で保存することもできます。暗号化、復号化(復号)、鍵管理の全てをAWS側で行なってくれます。（暗号化しないで保存も可能です）

    ・Amazon S3 Intelligent-Tiering (S3 Intelligent-Tiering) は Amazon S3 のストレージオプションの 1 つであり、パフォーマンスの低下や、オペレーション上のオーバーヘッドを発生させることなく、最もコスト効率の高いアクセス階層に自動的にデータを移動することで、コストを最小限に抑えるように設計されています。ブジェクトは、高頻度のアクセス用に最適化された階層と、低頻度のアクセス用に最適化された 2 つのアクセス階層に保存されます。オブジェクトごとのモニタリングおよび自動化にかかる月額料金を抑えるために、Amazon S3 では S3 Intelligent-Tiering のオブジェクトのアクセスパターンをモニタリングし、30 日間連続してアクセスがなかったオブジェクトは低頻度のアクセス階層に移動されます。低頻度のアクセス階層のオブジェクトにアクセスすると、データは自動的に高頻度のアクセス階層に再度移動されます。

    ・静的ウェブサイトを Amazon S3 でホスティングできます。静的ウェブサイトでは、個々のウェブページの内容は静的コンテンツです。ほかに、クライアント側スクリプトが含まれていることもあります。対照的に、動的ウェブサイトはサーバー側処理に依存しており、例えばサーバー側スクリプト（PHP、JSP、ASP.NET など）が使用されます。サーバー側スクリプトは Amazon S3 ではサポートされていません。

    ・Amazon S3 のバケットの中にバケットを作成することはできません。

  10. Question 10 of 11
    10. Question
    Category: Auto Scaling

    2つのインスタンスで Auto Scaling の スケーリングアクティビティ を開始しました。Auto Scaling グループのクールダウン期間は6分です。1つ目のインスタンスは3分後に立ち上がり、2つ目のインスタンスは4分後に立ち上がりました。何分後にオートスケールはスケーリングアクティビティのリクエストを許可しますか。最も正しいものを一つ選択してください。

    正解

    スケーリングアクティビティのリクエストを許可する時間は、グループ内の2つのインスタンスが起動してから、クールダウンの期間を経過した後になります。つまり、グループ内のインスタンスのローンチ後：4分 ＋ クールダウン期間：6分 の10分後に他のリクエストを受け付けるようになります。
    補足：
    スケーリングアクティビティ（Scaling Activity）
    スケーリングアクティビティは、グループのサイズを変えるなどのAuto Scalingへの変更を実装した、長期間動作するプロセスです。 それは、インスタンスを置き換えるとか、サービスによってサポートされた処理を長時間実行するプロセスだったりするかもしれません。

    不正解

  11. Question 11 of 11
    11. Question
    Category: 廃棄

    クラウドと従来のオンプレミス環境では、データを廃棄する方法が異なります。お客様が AWS にクラウド内のデータの削除を依頼した場合、AWS は基になる物理メディアを廃棄しません。代わりに、ストレージブロックが未割り当てとしてマークされます。AWS は安全なメカニズムを使ってブロックを他に再割り当てします。ブロックストレージをプロビジョニングする場合、ハイパーバイザーまたはVirtual Machine Manager (VMM) によって、インスタンスの書き込み先のブロックが追跡されます。インスタンスがストレージのブロックに書き込むときに、前のブロックはゼロで埋められた後、お客様のデータのブロックで上書きされます。お客様のインスタンスが以前に書き込んだブロックからの読み取りを試行した場合、以前に保存したデータが返されます。インスタンスが以前に書き込んでいないブロックからの読み取りを試行した場合の動作として正しいものを一つ選択してください。

    正解

    インスタンスが以前に書き込んでいないブロックからの読み取りを試行すると、ハイパーバイザーはディスクにある以前のデータをゼロで埋めて、インスタンスにゼロを返します。
    参考PDF：[AWS セキュリティのベストプラクティス]P52 データとメディアの安全な廃棄

    不正解

    インスタンスが以前に書き込んでいないブロックからの読み取りを試行すると、ハイパーバイザーはディスクにある以前のデータをゼロで埋めて、インスタンスにゼロを返します。
    参考PDF：[AWS セキュリティのベストプラクティス]P52 データとメディアの安全な廃棄
